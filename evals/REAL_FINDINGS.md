# Real-photo extraction eval: findings

This is the durable summary of the manual extraction eval (the live-model half).
The raw per-run output is `evals/real_results.md`, which is regenerated by
`python -m evals.real_eval` and is not committed because it changes every run and
is non-deterministic. The photos are local only (gitignored).

## What was tested

Five real bottles bought in the US, each photographed front and back, with both
faces sent in one model call so the model can read fields split across them:

- In scope (distilled spirits, should be compliant): Casamigos Tequila, Kirkland
  London Dry Gin, Baileys Irish Cream.
- Out of scope (boundary tests): a yogurt-flavored Soju (an edge spirit) and a
  Plum Sake (a fermented rice product regulated as wine under 27 CFR Part 4).

Each bottle was run on both `claude-haiku-4-5` and `claude-sonnet-4-6`.

## What it found

### 1. It corrected a real rule bug (high value)

Every in-scope bottle prints the Government Warning entirely in capital letters,
which is compliant: 27 CFR 16.22 requires only the words "GOVERNMENT WARNING" to
be capitalized, not the body. The original rule compared the whole warning
case-sensitively and false-FAILed all of them. The rule was corrected to compare
the wording ignoring case while still requiring the "GOVERNMENT WARNING" header in
capitals. The synthetic golden set could not have caught this, because every
mutant was built from the mixed-case canonical text. This is the headline result:
real data caught an assumption baked into a rule.

### 2. The deterministic rules are correct; perception is the variance

In every case the rule engine did the right thing given the fields it was handed.
All of the noise is in the model reading the label. The verbatim Government
Warning is the hardest field to read off a curved, small-print, low-contrast back
label, and it is where the failures cluster:

- Dropped the "GOVERNMENT WARNING:" header (Gin on Haiku, Sake).
- Dropped the spaces around punctuation, "WARNING:(1)ACCORDING" (Soju).
- Shortened wording, "alcoholic beverages" read as "alcohol" (Sake).

In all of these the model also reported `warning_legible = true`: it was
confidently wrong. So the model's self-reported legibility does not catch a
mis-transcribed warning, which means a verbatim warning check on photos cannot be
fully trusted to the model and belongs in front of a human.

### 3. Haiku vs Sonnet is not a clean win

Sonnet fixed the Gin header that Haiku dropped, but read the Tequila class/type as
just "Añejo" (routing it to review) and still mis-read the Baileys warning. It
costs about three times as much per label. Upgrading the model does not solve the
verbatim-warning problem; it moves it around. Recommendation: keep Haiku as the
default for cost, and treat the warning as human-confirmed rather than relying on
a model upgrade.

### 4. Out-of-scope beverages route to review, as they should

Soju and Sake both had their class/type correctly sent to NEEDS REVIEW. The tool
did not pretend to judge a beverage type it does not cover. Their overall FAILs
are scope artifacts (spirits rules applied to a wine or an edge spirit), not real
defects, and are labeled as such in the runner.

## Measured cost

About $0.006 per bottle on Haiku and about $0.017 on Sonnet, one call per bottle
across both photos. The full five-bottle Haiku run cost about three cents.

## Recommended follow-ups

1. Surface the warning diff (expected text vs the text the model read) on the
   result page, so a human reviewing a warning FAIL can tell in seconds whether it
   is a real defect or a transcription glitch. The data for this is already on the
   rule outcome (`expected_near` / `got_near`).
2. Keep `claude-haiku-4-5` as the default; offer Sonnet via `CLAUDE_MODEL` when a
   hard label warrants it.
3. Expand the recognized class/type set over time (for example tequila age
   statements) to reduce benign reviews, keeping the fail-safe that unknown terms
   review rather than fail.
